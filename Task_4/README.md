## Жадный алгоритм для задачи о распределении ресурсов
* **Определение:** это метод решения оптимизационных задач, который основан на локально наилучшем выборе на каждом этапе вычислений с надеждой, что этот выбор приведет к глобальному оптимальному решению. Ключевое свойство такого подхода состоит в том, что решение принимается немедленно и необратимо на каждом шаге.
* **Анализ алгоритма:**
    * Вычисление коэффициента эффективности: Алгоритм вначале рассчитывает эффективность (`profit/resource`), определяющую доходность вложений в каждый проект  (project["efficiency"] = project["profit"] / project["resource"])
    * Сортировка проектов: Проекты сортируются по убыванию эффективности, чтобы выбирать самые выгодные первым делом (sorted_projects = sorted(projects, key=lambda x: x["efficiency"], reverse=True))
    * Отбор проектов: Проект выбирается, если его включение не превысит лимит общего ресурса  (if current_resource + project["resource"] <= total_resource:)
    * Итоговая оценка прибыли: Подсчитывается общая прибыль всех выбранных проектов путём суммирования прибылей (total_profit = sum([project["profit"] for project in selected_projects]))
* **Временная сложность:** O(nlogn)
* **Объяснение временной сложности:** Временная сложность O(nlogn) появляется из-за этапа сортировки проектов по значению эффективности. Стандартные методы сортировки требуют порядка nlogn операций, что и определяет главную компоненту временной сложности алгоритма.
